Реализация реактивного манифеста на примере мониторинга Оружейной World of Warcraft.

Начав изучать онлайн курс на Pluralsight по созданию веб-сайтов реализующих реактивный манифест, я решил, что было бы неплохо параллельно применить полученные навыки на реальной задаче.

Важно: я не являюсь специалистом по созданию подобного рода приложений, как раз наоборот. Таким образом, обучение будет идти по мере создания проекта. Приветствуются конструктивные комментарии.

Реактивный манифест

Относительно новомодное явление в архитектуре построения веб-приложений, реактивный манифест обычно описывается, как инфраструктура веб-сайта реализующая четыре общих правила, т.е. она должна быть:

  • Responsive - интерактивной для пользователя
  • Scalable - автоматически масштабироваться по мере увеличения нагрузки
  • Resilient - устойчива ко сбоям
  • Event-driven - отвечать на асинхронные события от внешних источников

Используя эти принципы создадим веб-сайт для решения следующей задачи.

Контекст

Игра World of Warcraft имеет очень активный сектор Player vs Player (PvP) арен, на которых можно играть друг против друга командами 2х2, 3х3 или 5х5. Играющие команды ранжируются по системе аналогичной шахматному рейтингу ЭЛО. Раньше рейтинг этих команд можно было посмотреть по ссылке на Оружейную. Недавно этот рейтинг изменили и теперь вместо командного он стал личным.

В данном проекте cоздадим веб-сайт, который будет решать следующие задачи:

  • Онлайн мониторинг относительной силы классов в рейтинге Оружейной
  • С помощью машинного обучения попытаемся определять составы команд на основе личных показателей игроков

Используемые технологии

  • Язык программирования: С#
  • Платформа: Microsoft ASP .NET
  • Развертывание: Windows Azure и Amazon Web Services
  • База данных: MongoDB
